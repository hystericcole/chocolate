The Cole Color Model is like HSL with perceptual luminance. Using perceptual luminance means that as the hue or chroma change, the color will appear to have a consistent luminance. Compare this to HSL, where blues and yellows with the same lightness have drastically different luminance.

The H in CHCLT, like HSL or LCH, is hue. Hue is a measure of the ratio of primary colors. Like LCH, the primary colors are at mathematically significant positions in the hue range, not evenly distributed like HSL. CHCLT uses red as a reference color for hue, so like HSL red is at both ends of the hue range.

The middle C in CHCLT is chroma, which is similar to saturation in HSL. Higher chroma creates a more intense, saturated color. Lower chroma creates a less intense, desaturated color. Pure grays have zero chroma, and fully saturated colors like primary colors have a chroma of one.

The L in CHCLT is luma. Luma is related to the luminance of the color, and equals the gray value of a color with zero chroma. Black has luma zero and white has luma one. Luma can be used to easily find colors that are legible in contrast to other colors.

This page uses sRGB as the underlying color space for CHCLT to transform, but CHCLT can be used with any RGB color space that has a reversible transfer function and luminance function, e.g. any color space that can be converted to and from CIEXYZ.

The T in CHCLT is transform. This color model started as a series of transforms on RGB colors, and did not become a color model until operations on hue were defined.

The color sliders above have ranges between 0 and 100 (or 360 for hue) but the underlying math for CHCLT always uses a range of zero to one for all components.

Hue is cyclic, so any value for hue is valid. Luma ranges from zero for black to one for white, with all the colors between. Chroma ranges from zero for gray to one for maximum saturation in gamut. Negative chroma values will create a complementary color. Chroma values with a magnitude exceeding one will create an RGB color that is out of gamut, but may be valid in some contexts.

True perceptual uniformity is difficult. CIACAM02 is a good example of how difficult. CHCLT has a perceptual luminance based on the underlying color space, which is usually a good approximation. The Lab color space provides a better approximation, but it is difficult to stay in gamut while operating in the Lab color space.

Luma makes it easy to find contrasting colors. There are standards like WCAG G17 and G18 that define the luminance based contrast between colors using an offset ratio of luminances. Colors with a sufficient ratio are considered more legible and satisfy accessibility guidelines. With luma, the ratio isn't needed. Just use a color with a luma that differs by at least 0.5 (for G18) or 0.6 (for G17) to create a contrasting color. The text examples use 0.5 as the luma difference.

Gradients in RGB color space tend to go through gray and tend to have inconsistent luminance. Using CHCLT to define the color stops of a gradient allow the gradient to follow a curve through RGB space and have predictable luminance. That curve can be similar to the gradients created by the cube helix method.

Conversion to and from RGB is computationally expensive compared to color models like HSL or HSV. The Cole color model is well suited to choosing the colors for a theme or palette in a user interface, but not to image processing.

There are 3 spaces that the colors move through. The RGB space, or display space. The linear space, defined by the inverse transfer function of the underlying color space, where multiplication and addition are well defined.  And the abstract lightless space where luminance has been factored out and angles are between hues.

				
Luma is defined in display space, and relates directly to luminance in linear space.  Hue is in lightless space.  Chroma is in lightless space and also depends on luminance in linear space.
